# 小事清 - 测试文档 (test.md)

## 1. 文档说明
- **版本**：v1.0
- **更新日期**：2026-05-09
- **测试范围**：小程序前端 + 云函数后端 + 微信生态集成
- **测试阶段**：单元测试、集成测试、端到端测试、验收测试

## 2. 测试策略

### 2.1 测试层级
| 层级 | 范围 | 工具/方法 | 负责人 |
|------|------|-----------|--------|
| 单元测试 | 云函数核心逻辑（伪代码验证） | 手工 code review + 云函数本地模拟 | 后端开发 |
| 集成测试 | 云函数 + 数据库 + 云存储 | 微信开发者工具 + 真机调试 | 测试工程师 |
| 端到端测试 | 完整用户场景（前端+后端） | 真机（iOS/Android） | 测试工程师 + 内测用户 |
| 验收测试 | 满足需求和产品目标 | UAT（用户接受测试） | 产品经理 + 种子用户 |

### 2.2 测试环境
- **开发环境**：微信开发者工具（Stable 1.06.2409140+），云开发环境 ID：`xiao-shi-qing-dev`
- **测试环境**：独立云开发环境 `xiao-shi-qing-test`，数据库、云存储、云函数均隔离
- **生产环境**：`xiao-shi-qing-prod`

### 2.3 测试数据
- 使用独立的测试 openid（mock 多个用户）
- 预置数据：不同状态、标签、耗时的小事记录，含照片/录音

## 4. 非功能测试

### 4.1 性能测试

| ID | 测试项 | 指标 | 测试方法 | 通过标准 |
|----|--------|------|----------|----------|
| PT-01 | 首页启动时间 | ≤2秒 | 真机录制，4G网络 | 从点击图标到可交互小于2秒 |
| PT-02 | 语音转文字延迟 | ≤3秒 | 按住说话松手后计时 | 90%请求≤3秒 |
| PT-03 | 照片上传成功率 | >99% | 弱网模拟（限速） | 上传失败后自动重试，最终成功率＞99% |
| PT-04 | 清单分页加载 | 20条<500ms | 列表滚动加载 | 平均耗时<500ms |
| PT-05 | 导出PDF耗时 | 含10条记录<10秒 | 点击导出到完成 | 正常网络下<10秒 |

|----|--------|------|----------|

## 5. 异常场景测试

| ID | 场景 | 操作 | 预期处理 |
|----|------|------|----------|
| ET-01 | 网络断开时保存 | 飞行模式下手动录入保存 | 提示“网络不可用，已暂存至本地”，联网后自动同步 |
| ET-02 | 云函数超时 | 大量数据导出 | 提示“生成较慢，请稍后查看”，后台异步生成，通过服务通知推送 |
| ET-03 | 照片上传失败 | 弱网下添加照片 | 显示上传失败图标，自动重试3次，仍失败提示用户重试 |

| ET-05 | 服务通知订阅失败 | 用户拒绝订阅 | 不再弹出引导，日后可手动在“我的”页点击“开启提醒” |

## 6. 验收测试标准

### 6.1 功能完整性
- 所有 P0 用例通过率 100%
- 所有 P1 用例通过率 ≥ 95%
- 所有 P2 用例通过率 ≥ 80%

### 6.2 非功能达标
- 性能指标 PT-01~PT-04 均达标
- 兼容性 CT-01~CT-04 无阻塞问题
- 安全测试 ST-01~ST-03 通过

### 6.3 用户验收测试（UAT）
- 邀请 10 名目标用户（门店店员、外卖员等）使用 3 天
- 收集反馈满意度 ≥ 4.0/5.0
- 至少 8 人认为“解决了我记录破事的需求”

## 9. 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 语音转文字准确率低导致用户流失 | 提供手动编辑入口，增加方言优化选项 |
| 真机兼容性问题（尤其Android） | 提前使用云真机测试，覆盖主流品牌 |
| 服务通知被微信限制 | 遵循微信规范，不滥用，用户主动订阅 |
| 证据包生成耗时长 | 异步生成+通知领取，避免阻塞 |
